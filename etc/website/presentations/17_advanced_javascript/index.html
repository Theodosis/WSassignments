Location

  • Η διέυθυνση στην οποία βρισκόμαστε
  • Περιέχει τη συνάρτηση reload
  • Περιέχει τις ιδιότητες href, protocol, hostname, pathname, hash.
    • window.location.href == "http://myblog.com/posts/13/#comment-22"
      • window.location.protocol == "http:"
      • window.location.hostname == "myblog.com"
      • window.location.pathname == "/posts/13/"
      • window.location.hash == "#comment-22"

toString function

  • Όταν η JavaScript χρειάζεται να μετατρέψει ένα αντικείμενο σε string ψάχνει τη συνάρτηση toString σε αυτό το αντικείμενο.
  • Αν τη βρει, την εκτελεί και χρησιμοποιεί το αποτέλεσμά της
  • Αλλιώς κάνει ότι νομίζει
var person = {
    firstname: 'John',
    lastname: 'Smith',
    conduct: 'decorous',
    toString: function(){
        var result = this.firstname + ', ' + this.lastname;
        if( this.conduct == 'decorous' ){
            return result + ' the decorous one';
        }
        return result;
    }
};
alert( person );

HTML

<html>
    <body>

        </p>

  • Κάθε HTML στοιχείο αναπαριστάται με ένα μοναδικό αντικείμενο
  • Υπάρχουν αντικείμενα DOM που δεν αντιστοιχούν σε στοιχεία HTML
  • Nodes

    • Τα HTML σχόλια είναι Comment Nodes

DOM Traversing

  • Σε κάθε Node έχουμε μεθόδους και ιδιότητες για να διασχίσουμε το DOM δέντρο
  • Με αυτό τον τρόπο βρίσκουμε το στοιχείο που θέλουμε
  • Ως βάση έχουμε το document.body
  • Η πληθικότητα των αποτελεσμάτων καθορίζει και τον τύπο δεδομένων τους
    • Σε συναρτήσεις ή ιδιότητες που μπορούν να επιστρέψουν πολλά στοιχεία, επιστρέφεται πίνακας
    • Σε συναρτήσεις ή ιδιότητες που μπορούν να επιστρέψουν μόνο ένα στοιχείο, επιστρέφεται Node
  • Το όνομα της συνάρτησης/ιδιότητας προδίδει και την πληθικότητα των αποτελεσμάτων της

DOM Traversing

  • x.childNodes
    • Είναι ένας πίνακας
    • Περιέχει όλα τα παιδιά του στοιχείου x
    • Το πρώτο παιδί είναι το στοιχείο 0
    • Το μήκος του είναι x.childNodes.length
  • x.parentNode
    • Περιέχει τον πατέρα του x
    • Είναι σίγουρα Element Node
  • x.nextSibling
    • Περιέχει το επόμενο Node από τα παιδιά του πατέρα του x
  • x.previousSibling
    • Περιέχει το προηγούμενο Node από τα παιδιά του πατέρα του x

  • Μέθοδοι σε κάθε Element Node
  • Αντί για μεγάλες, δύσχρηστες αλυσίδες από childNodes[ ].childNodes[ ] ...
  • Πηγαίνουμε κατευθείαν στο στοιχείο ανεξάρτητα από τη δομή του δέντρου

DOM functions

  • x.getElementById( id )
    • Επιστρέφει το στοιχείο που έχει id και είναι απόγονος του x
    • Έχει πληθικότητα 1

    • Επιστρέφει πίνακα με τα αποτελέσματα του css επιλογέα, όταν εκτελεσθεί για τους απογόνους του x

Events

  • Χειριστές (handlers)
    • Συναρτήσεις που τρέχουν όταν γίνεται κάποιο γεγονός
  • Γεγονότα (events)
    • Συμβαίνουν όταν αλληλεπιδρά ο χρήστης με τη σελίδα
    • Αφορούν κάθε φορά ένα στοιχείο
  • Αντιστοιχίζουμε χειστές με γεγονότα
  • Όταν συμβεί το γεγονός τρέχει ο χειριστής
  • Τέσσερις μεγάλες κατηγορίες από events
    • Mouse Events
    • Keyboard Events
    • Browser Events
    • Server Events

jQuery events

  • x.eventName();
    • Χωρίς ορίσματα πυροδοτεί το event στα επιλεγμένα elements
    • Με 1 όρισμα τύπου συνάρτηση, αναθέτει στο event των επιλεγμένων elements τη συνάρτηση αυτή ως handler
    • Όταν πυροδοτηθεί το event (είτε από το χρήστη είτε από τη javascript), θα δωθεί στον handler ένα αντικείμενο τύπου event

jQuery event object

  • Διαφορετικό για διαφορετικά events

default behaviour

this & event handlers

  • Σε μια συνάρτηση χειριστή, η λέξη κλειδί this αναφέρεται στο html element στο οποίο δημιουργήθηκε το event

Optional parameters

    • Τα ορίσματα που περνάνε σε μια συνάρτηση JavaScript όταν την καλούμε μπορεί να είναι περισσότερα απ' όσα ορίζονται.
    • arguments: Πίνακας που περιέχει τα ορίσματα που έχουν περάσει στη συνάρτηση
    function add(){
    var length = arguments.length;
    var sum = 0;
    for( var i = 0; i < length; ++i ){
        sum += arguments[ i ];
    }
    return sum;
}
alert( add( 1, 2, 3 ) );
alert( add( 6, 21, -5, 7, 0 ) );

Τοπικές και καθολικές μεταβλητές

  • Οι καθολικές μεταβλητές είναι προσβάσιμες από παντού
  • Ορίζονται με var εκτός όλων των συναρτήσεων
  • Αποτελούν ιδιότητες στο window
  • Η JavaScript θεωρεί οτι αναφερόμαστε σε καθολική μεταβλητή όταν δεν την ορίζουμε
  • Οι τοπικές μεταβλητές είναι προσβάσιμες μόνο μέσα στη συνάρτηση που ορίζονται
  • Ορίζονται με var εντός της συνάρτησης
  • Τα ορίσματα της συνάρτησης, καθώς και ο πίνακας attributes είναι τοπικές μεταβλητές

function foo(){
    
    var a;
    
    a = 6;
    alert( a );
}
foo();
alert( a );

Recursion

  • Κλήση μιας συνάρτησης μέσα από τον εαυτό της (αναδρομή)
function factorial( n ){
    if( n == 1 ){
        return 1;
    }
    return factorial( n - 1 ) * n;
}
alert( factorial( 3 ) );

function makeMultiplier( factor ){
    var multiplier = function( x ){
        return x * factor;
    };
    return multiplyer;

alert( double( 3 ) );

closures

  • Οι συναρτήσεις που ορίζονται μέσα σε άλλες συναρτήσεις έχουν πρόσβαση στις μεταβλητές των εξωτερικών συναρτήσεων
  • Αυτό ονομάζεται κλείσιμο (closure)
var makeMultiplier = function( factor ){
    var multiplier = function( x ){
        return factor * x;
    }
    return multiplier;
}
